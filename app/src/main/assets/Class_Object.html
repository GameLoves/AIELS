Classes and Objects in C#

Introduction to Classes and Objects

A class is a blueprint or template for creating objects, which are instances of the class. In C#, a class defines the data and behavior of an object. Here's an example:


public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public void SayHello()
    {
        Console.WriteLine("Hello, my name is " + Name + " and I am " + Age + " years old.");
    }
}

Person john = new Person();
john.Name = "John";
john.Age = 30;
john.SayHello(); // writes "Hello, my name is John and I am 30 years old." to the console

In this example, we define a class named Person with two properties named Name and Age, and a method named SayHello. We then create an object of the Person class named john, set its properties, and call its SayHello method.

Types of Classes

In C#, there are different types of classes based on their accessibility and behavior. Here are some of the commonly used types of classes:

  • Public class: A public class is accessible from anywhere in the code and can be used by other classes and objects.
  • Private class: A private class is only accessible within the same file or class and cannot be used by other classes and objects.
  • Abstract class: An abstract class is a class that cannot be instantiated and is meant to be used as a base class for other classes. It can contain abstract and non-abstract methods.
  • Sealed class: A sealed class is a class that cannot be inherited by other classes.
  • Static class: A static class is a class that cannot be instantiated and contains only static members such as fields, properties, and methods.

Access Modifiers

In C#, access modifiers are used to control the access to the members of a class. Here are the access modifiers that can be used with classes:

  • Public: A public class is accessible from anywhere in the code.
  • Internal: An internal class is accessible only within the same assembly (or project).
  • Protected: A protected class is accessible only within the same class or its derived classes.
  • Private: A private class is accessible only within the same class.
  • Protected internal:A protected internal class is accessible within the same assembly (or project) or its derived classes, regardless of the assembly they are in.

Conclusion

Classes and objects are fundamental concepts in C#. A class defines the blueprint for creating objects, and an object is an instance of a class. You can define different types of classes based on their accessibility and behavior. Access modifiers are used to control the access to the members of a class. Understanding these concepts is essential for creating robust and maintainable C# code.
