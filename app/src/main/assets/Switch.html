Switch Statements in C#

The switch statement is used to execute one of several possible blocks of code, based on the value of a given expression. Here's the basic syntax of a switch statement:


switch (expression)
{
    case value1:
        // code to execute if expression == value1
        break;
    case value2:
        // code to execute if expression == value2
        break;
    // more cases here...
    default:
        // code to execute if none of the above cases match
        break;
}

The expression inside the parentheses is evaluated, and then the code inside the first matching case block is executed. If none of the case blocks match, the code inside the default block is executed. Here's an example:


int dayOfWeek = 3;

switch (dayOfWeek)
{
    case 1:
        Console.WriteLine("Monday");
        break;
    case 2:
        Console.WriteLine("Tuesday");
        break;
    case 3:
        Console.WriteLine("Wednesday");
        break;
    case 4:
        Console.WriteLine("Thursday");
        break;
    case 5:
        Console.WriteLine("Friday");
        break;
    default:
        Console.WriteLine("Weekend");
        break;
}

In this example, the value of dayOfWeek is 3, so the third case block is executed, and the output is "Wednesday".

You can also use the switch statement with other data types, including strings and enums. Here's an example with a string:


string fruit = "apple";

switch (fruit)
{
    case "apple":
        Console.WriteLine("This is an apple.");
        break;
    case "banana":
        Console.WriteLine("This is a banana.");
        break;
    case "orange":
        Console.WriteLine("This is an orange.");
        break;
    default:
        Console.WriteLine("I don't know what this fruit is.");
        break;
}

In this example, the value of fruit is "apple", so the first case block is executed, and the output is "This is an apple."

These are just a few examples of what you can do with switch statements in C#. There are many more ways to use them, and you can also combine them with other programming constructs to create more complex logic.
